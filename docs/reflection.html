Comparative Reflection: GitHub Pages vs. WordPress

Introduction

Over the course of this project, I built the same personal portfolio website using two very different platforms: GitHub Pages (using hand-coded HTML and CSS) and WordPress through Penn State's Sites at Penn State (sites.psu.edu). Both approaches produced a working website, but the experience of building each one was radically different. This reflection compares those two experiences and examines what each method demands from a new learner.

You can view my alternate website here: sites.psu.edu/digit100deb/

Biggest Challenges for a New Learner

The biggest challenge with GitHub Pages was the learning curve around the tools themselves — not just HTML and CSS, but also Git, the terminal, and understanding how version control works. Before you can even see your website live, you have to understand concepts like repositories, commits, branches, and pushing to a remote. For someone with no programming background, this is a significant barrier. A simple typo in a file path means your images don't load and you have to track down why.

WordPress, on the other hand, was much easier to get started with. The dashboard is visual and familiar — it resembles tools like Google Docs or social media content editors. The biggest challenge with WordPress was not technical, but creative: figuring out how to organize your content within the constraints of the theme and block editor, and understanding the difference between Posts and Pages. There was less troubleshooting but also less flexibility.

Advantages of Each Method

GitHub Pages gives you total control. Every pixel of the site is the result of code you wrote. This means you can build exactly what you envision — custom layouts, unique styling, interactive elements — without being limited by a theme or a plugin. It also teaches real, transferable skills: HTML, CSS, Git, and the command line are used by professional developers every day. For someone studying Marketing and MIS, understanding how web technology works under the hood is genuinely valuable.

WordPress's advantage is speed and accessibility. You can have a professional-looking website up in minutes without writing a single line of code. It has built-in features for navigation menus, pages, media libraries, and themes that would take hours to build from scratch. For small businesses, bloggers, or anyone who needs a functional website quickly, WordPress is clearly the more practical tool.

Challenges for Modifying Layout and Organization

Modifying the layout in GitHub Pages means editing the HTML and CSS directly. Want to move a navigation bar? Edit the HTML. Want to change font sizes across the whole site? Update the CSS. This gives precise control but requires knowledge of how those languages work and how changes in one file affect others. It also means that updating navigation on multiple pages requires editing each file individually — there's no shared template unless you build one yourself.

In WordPress, layout changes are handled through the Customizer and block editor, which are visual tools. However, making deeper structural changes — like modifying how the header works or changing the column layout — often requires either switching themes or writing custom CSS, which can feel frustrating when you assumed no coding was needed. The theme constrains what you can easily change, and sometimes what you want simply isn't possible without going deeper into the platform.

Skills Required for Each Platform

To build a site with GitHub Pages, you need HTML and CSS for structure and styling, Git and the terminal for version control and deployment, a code editor (like Oxygen XML Editor or VS Code) for writing files, and a basic understanding of file systems and relative paths. These are technical skills that take time to develop but are widely applicable in technology careers.

For WordPress, the core skills are more about content strategy and familiarity with the platform's interface. Understanding how to use the block editor, manage pages vs. posts, configure menus, and select and customize themes are the key competencies. Basic CSS knowledge is helpful for customization, but not required to get started. The platform rewards organizational thinking over technical coding ability.

Skills I Want to Learn to Improve My Websites

For my GitHub Pages site, I would like to learn more CSS — specifically Flexbox and Grid for building cleaner, more responsive layouts. I'd also like to learn some basic JavaScript to add interactivity, like a responsive navigation menu or animated transitions. Understanding how to use a static site generator like Jekyll would also help me scale the site without having to update the navigation on every page manually.

For my WordPress site, I'd like to explore more advanced theme customization and learn how to use custom CSS effectively within the Customizer. I'm also interested in learning how to use WordPress as a portfolio platform more intentionally — organizing projects, embedding media, and using plugins to add features like contact forms or image galleries.

Overall, this project gave me a strong foundation in both approaches to web development. As someone studying Marketing and MIS, I can see clear professional use cases for both: GitHub Pages for technical portfolio work and personal branding in tech-adjacent fields, and WordPress for building client-facing websites quickly and without requiring a developer. Both skills are worth developing further.
